## 1. 기획 방향

### 1.1 기존 vs 신규 비교

| 구분 | 기존 랜딩 | 신규 랜딩 |
|------|-----------|-----------|
| **타겟** | 예비창업자/기창업자 | 기관담당자 + 멘토 + 창업자 |
| **메시지** | "AI로 사업계획서를 만들자" | "대한민국 창업지원 생태계를 하나로 연결합니다" |
| **구조** | 단일 흐름 (트랙 선택) | 이해관계자별 가치 제안 + 통합 워크플로우 |
| **CTA** | 회원가입 1개 | 역할별 CTA 3개 (기관등록 / 멘토지원 / 창업시작) |
| **신뢰요소** | 통계, 후기 | 참여기관 로고, 정부사업 연계, 실적 데이터 |

### 1.2 설계 원칙

1. **3초 안에 "누구를 위한 서비스인지" 인식** — 히어로에서 3대 이해관계자 명확히 표시
2. **역할별 가치를 빠르게 탐색** — 스크롤 또는 탭으로 자신의 역할 섹션 바로 이동
3. **하나의 플랫폼 안에서 연결되는 느낌** — 워크플로우 다이어그램으로 전체 흐름 시각화
4. **신뢰감 확보** — 정부 창업지원사업 연계, 참여기관 로고, 보안/개인정보 안내

### S1. 히어로 (HeroSection)

**목표:** 첫 화면에서 "3대 이해관계자를 위한 통합 플랫폼"임을 전달

**레이아웃:**
```
[배지] 모두의 창업 × CASA

[메인 카피]
대한민국 창업 지원,
하나의 플랫폼에서 완성하다

[서브 카피]
기관의 행정 부담은 줄이고, 멘토의 지도는 체계적으로,
창업자의 아이디어는 AI로 빠르게. CASA가 연결합니다.

[3개 역할 카드 — 가로 배치]
┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│ 🏛️ 기관담당자 │ │ 🎓 멘토      │ │ 🚀 창업자    │
│             │ │             │ │             │
│ 지원자 현황   │ │ 체계적 멘토링  │ │ AI 문서 생성  │
│ 실시간 모니터링│ │ & 이력 관리   │ │ & 피드백 반영  │
│             │ │             │ │             │
│ [기관 등록 →] │ │ [멘토 지원 →] │ │ [창업 시작 →] │
└─────────────┘ └─────────────┘ └─────────────┘

[신뢰 지표] ✓ 정부 창업지원사업 연계  ✓ AI 기반 자동화  ✓ 보안 인증
```

**인터랙션:**
- 각 역할 카드 호버 시 카드 확대 + 배경색 변화
- 카드 클릭 시 해당 역할의 상세 섹션(S4)으로 스크롤
- 모바일: 카드를 세로 스택으로 전환

### S2. 문제 제기 (ProblemSection)

**목표:** 현재 창업지원 현장의 공감되는 문제를 3가지 관점으로 제시

**레이아웃:**
```
[섹션 제목] 창업 지원 현장, 이런 어려움 겪고 계신가요?

[3열 문제 카드]

기관담당자의 고충               멘토의 고충                 창업자의 고충
─────────────────         ─────────────────        ─────────────────
• 수백 명 지원자 관리에         • 여러 플랫폼에 분산된        • 사업계획서 작성에
  엑셀과 메일만 반복             멘토링 기록                  몇 주씩 소요
• 멘토 매칭 수기 처리           • 지원자 진행상황 확인         • 멘토 피드백 반영이
• 수당 지급 서류 정리             어려움                      비효율적
  업무 과부하                  • 의견서 작성에 시간 소모       • 어디서부터 시작할지 막막
```

**디자인:**
- 각 카드에 pain point를 상징하는 일러스트 또는 아이콘
- 빨간/주황 톤의 경고성 색상 → S3에서 파란/녹색 솔루션 톤으로 전환

### S3. 솔루션 개요 (SolutionOverviewSection)

**목표:** CASA가 3대 이해관계자를 어떻게 연결하는지 한 눈에 보여주기

**레이아웃:**
```
[섹션 제목] CASA, 창업 생태계를 하나로 연결합니다

[중앙 다이어그램]

       ┌─────────┐
       │  CASA   │
       │ Platform│
       └────┬────┘
            │
    ┌───────┼───────┐
    ▼       ▼       ▼
┌──────┐ ┌──────┐ ┌──────┐
│ 기관  │ │ 멘토  │ │ 창업자 │
│      │←→│      │←→│      │
└──────┘ └──────┘ └──────┘

기관이 멘토를 매칭하고 → 멘토가 피드백하고 → 창업자가 AI로 발전시키는
하나의 순환 구조
```

**내용:**
- 짧은 한 줄 설명 3개:
  - "기관은 지원자 현황을 실시간으로 파악하고 멘토를 매칭합니다"
  - "멘토는 AI가 생성한 문서를 검토하고 전문 피드백을 남깁니다"
  - "창업자는 멘토 의견을 반영해 AI로 빠르게 문서를 개선합니다"

**디자인:**
- 애니메이션: 스크롤 시 다이어그램이 단계별로 나타남
- 연결선에 데이터 흐름 애니메이션 (점이 이동하는 효과)

### S5. 워크플로우 (WorkflowSection)

**목표:** "모두의 창업" 프로그램이 CASA에서 어떻게 운영되는지 전체 흐름 시각화

```

STEP 1          STEP 2          STEP 3          STEP 4          STEP 5
가입 & 승인  →   매핑 & 매칭  →   AI 빌딩    →    멘토링 루프  →   완료 & 보고

승인, 멘토가    기관에 배정,     사업계획서,     피드백 →       생성, 기관

### S10. FAQ (FAQSection)

**목표:** 역할별 자주 묻는 질문 정리

**기존 컴포넌트 확장 — 카테고리 탭 추가:**

```
[카테고리: 전체 | 기관 | 멘토 | 창업자]

Q. 기관 등록은 어떻게 하나요?
A. 회원가입 시 "기관담당자"를 선택하고 소속 기관을 지정하면
   시스템 관리자 승인 후 이용할 수 있습니다.

Q. 멘토 수당은 어떻게 지급되나요?
A. 멘토링 완료 후 AI 의견서가 생성되면, 기관 담당자가 확인 후
   수당 지급을 승인합니다. 실제 지급은 기관 내부 절차를 따릅니다.

Q. AI가 만든 문서를 직접 수정할 수 있나요?
A. 네. AI 생성 문서를 직접 편집하거나, "OO 부분을 수정해줘"라고
   AI에게 지시하여 수정할 수 있습니다.

Q. 개인 프로젝트도 만들 수 있나요?

**신규 Nav:**
