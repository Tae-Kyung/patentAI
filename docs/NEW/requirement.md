나는 현재 개발되어 있는 CASA 프로젝트를 확장하고자 한다.
현재 한국에서는 모두의 창업이라는 대규모 창업지원 프로젝트가 진행될거야. 전국에서 창업을 희망하는 참가자들이 K-Startup 사이트에 신청서를 내고 각 지역(충북, 충남, 경북, 경남 등)의 창조경제센터와 창업중심대가 참가자들의 창업을 지원하는 역할을 하게 돼. 창조경제센터와 창업중심대에서는 멘토링을 통해 사업신청자들의 아이디어와 비즈니스 모델을 구체화하게 할 것이야. 그런데 지금 이러한 사업을 시스템없이 운영하는 것에 대한 걱정을 하고 있어. 그래서 나는 기존의 CASA 시스템을 확장해서 기관담당자(지역의 창조경제센터, 창업중심대), 멘토, 창업 지원자들이 이 시스템에서 모두의 창업 프로그램을 운영할 수 있도록 확장할거야. 회원은 시스템 관리자, 기관담당자, 일반가입자, 멘토로 가입을 할 수 있어.
- 시스템 관리자는 사전에 기관을 시스템에 등록해 두고 기관 담당자 가입시 기관을 선택해서 가입하면 관리자는 승인을 해주면 될 것 같아. 그리고 시스템 관리자는 일반 가입자 (즉 기관에서 관리해야할 모두의 창업자) 프로젝트를 기관으로 매핑해서 관리할 수 있도록 해주고, 기관 담당자는 자신들이 관리해야할 지원자들을 볼 수 있고 멘토들을 매칭해주는 역할을 할거야. 기관에서는 어느 프로젝트를 누가 멘토링 할 수 있는지 확인할 수 있고 기관별 지원자들의 아이디어 구체화 과정을 한눈에 볼 수 있게 돼. 
- 일반 가입자는 모두의 창업 지원자로서 자신의 아이디어를 기존 CASA 시스템을 이용하여 생성하고 피드백하면 되는거야. 프로젝트를 여러개 만들 수 있는데 그 중에서 기관에서 지원해주는 프로젝트와 개인 프로젝트로 구분을 할 수 있어. 또는 프로젝트를 만들어서 기관지원을 신청하면 기관은 승인후 관리도 할 수 있어야해. 
- 그리고 멘토는 로그인 하면 자신에게 할당된 지원자들의 프로젝트가 보이게 되고, 프로젝트에 들어가서 아이디어, 사업 계획서, 발표자료, 피칭자료, 포스터, 비즈니스 모델 등 CASA에서 만들어준 내용들을 검토하고 의견을 달아줄 수 있게 돼. 이후 프로젝트를 생성한 창업자는 멘토의 의견을 기반으로 자기의 아이디어를 좀 더 개선할 수 있어. 이렇게 진행된 과정을 히스토리로 남겨야 해. 이렇게 프로세스가 끝나면 멘토의견서와 최종 산출물이 만들어지고 제출을 하면 이 의견서를 확인한 뒤 기관 담당자는 멘토에게 수당을 지급할 수 있는거지. 
- 멘토 가입자의 경우 회원가입 후 별서의 설정 텝에서 자기의 이력서와 통장사본 등을 업로드할 수 있게 해주면 좋을거야. 
- 시스템 관리자는 기관별 전체 현황을 볼 수 있고, 기관-예비창업자 매칭을 해주는 역할을 하고, 기관은 할당된 프로젝트를 확인하고 프로젝트별 멘토를 매칭시켜주며, 예비창업자와 멘토간의 진행상황을 확인할 수 있어. 그리고 멘토링이 끝난 프로젝트에 대한 확인 및 멘토에게 수당 지급을 위한 업무를 여기에서 할거야. 


1. 제품 개요 (확장)
•	제품명: CASA (Center for AI-Agentic Startup Architecture) - 모두의 창업 에디션
•	핵심 가치: 1. 지원자: AI를 통한 초고속 사업 아이디어 구체화 및 문서 생성.
2. 멘토: 시스템화된 검토 환경과 데이터 기반의 정교한 멘토링.
3. 기관: 대규모 지원자의 실시간 현황 모니터링 및 행정 자동화(멘토링 보고서/수당 지급).
4. 관리자: 전국 단위 기관-지원자-멘토 매핑 및 거버넌스 관리.
2. 사용자 역할 및 권한 (신규 정의)

기관 담당자	관할 지원자 리스트 조회, 멘토-프로젝트 매칭, 멘토 수당 지급 관리	기관 증명서

3. '모두의 창업' 특화 워크플로우 (Human-Mentor-AI Loop)
1.	가입 및 승인: 멘토는 가입 시 이력서와 통장사본을 업로드하며, 시스템 관리자가 기관을 승인합니다.
2.	매핑 및 매칭: 관리자가 지원자를 특정 기관에 매핑하면, 기관 담당자가 내부 멘토 풀(Pool)에서 적합한 멘토를 매칭합니다.
3.	AI 빌딩 (지원자): 기존 CASA 기능을 사용하여 사업계획서, BM, 포스터 등을 생성합니다.
4.	멘토링 (멘토): 멘토 대시보드에서 지원자의 산출물을 실시간 검토하고, AI가 생성한 각 섹션에 '멘토 코멘트'를 남깁니다.
5.	피드백 반영: 지원자는 멘토 의견을 확인하고 AI에게 "멘토 의견을 반영하여 사업계획서 3절을 수정해줘"라고 지시합니다.
6.	최종 승인 및 리포트: 멘토링이 완료되면 **'AI 멘토 의견서'**가 자동 생성됩니다.
7.	행정 처리: 기관 담당자가 의견서를 최종 확인하면 멘토의 통장사본 정보와 연동되어 수당 지급 프로세스로 넘어갑니다.
4. 데이터 모델 확장 (ERD 수정)
기존 스키마에 기관, 멘토 정보, 매칭 이력을 관리하는 테이블이 추가되어야 합니다.
SQL
-- 기관 정보 테이블
CREATE TABLE bi_institutions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  region TEXT, -- 전국 창조경제혁신센터/대학 지역
  is_approved BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 멘토 프로필 (추가 정보)
CREATE TABLE bi_mentor_profiles (
  user_id UUID PRIMARY KEY REFERENCES bi_users(id),
  resume_url TEXT, -- 이력서 파일 경로
  bank_account_url TEXT, -- 통장 사본 파일 경로
  bank_name TEXT,
  account_number TEXT,
  specialty TEXT[], -- 전문 분야
  is_active BOOLEAN DEFAULT true
);

-- 멘토-프로젝트 매칭 테이블
CREATE TABLE bi_mentor_matches (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID REFERENCES bi_projects(id),
  mentor_id UUID REFERENCES bi_users(id),
  institution_id UUID REFERENCES bi_institutions(id),
  status TEXT DEFAULT 'assigned' CHECK (status IN ('assigned', 'in_progress', 'completed')),
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 멘토링 리포트 및 수당
CREATE TABLE bi_mentoring_reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  match_id UUID REFERENCES bi_mentor_matches(id),
  summary TEXT, -- 멘토의 최종 의견 요약
  ai_summary TEXT, -- AI가 정리한 멘토링 성과
  payout_status TEXT DEFAULT 'pending' CHECK (payout_status IN ('pending', 'approved', 'paid')),
  created_at TIMESTAMPTZ DEFAULT now()
);
5. 주요 화면 기능 개선
5.1 기관 담당자용 통합 대시보드
•	실시간 진행률: 관할 구역 지원자 100명의 AI 문서 생성 현황(아이디어-평가-문서-배포)을 한눈에 파악.
•	멘토 매칭 매트릭스: 프로젝트 분야(예: 헬스케어)와 멘토의 전공분야를 AI가 분석하여 '추천 멘토' 제시.
•	수당 지급 리스트: 멘토 의견서가 완료된 건들을 모아 일괄 지급 승인.
5.2 멘토용 '워크스테이션'
•	CASA 뷰어: 지원자가 만든 사업계획서 마크다운 문서를 좌측에 띄우고, 우측에 코멘트 창 배치.
•	문서별 검토: 사업계획서뿐만 아니라 AI 생성 포스터, 비즈니스 모델 캔버스, 피칭 스크립트를 개별 탭으로 검토 가능.
•	이력 관리: 가입 시 업로드한 서류 수정 및 매칭 이력 관리.
5.3 일반 지원자용 '멘토링 루프'
•	수정 요청 에이전트: "멘토님이 기술적 차별점이 부족하다고 하시는데, 우리 솔루션에 양자 컴퓨팅 기술을 접목해서 차별점을 강화해줘"와 같은 명령 수행.

1.	K-Startup 연동성: 향후 K-Startup의 신청서 데이터(JSON/PDF)를 CASA로 바로 불러올 수 있는 '가져오기' 기능을 추가하면 유입률이 폭발적으로 늘어날 것입니다.
2.	멘토링 품질 평가 AI: 멘토가 대충 의견을 남기지 않도록, AI가 멘토의 코멘트 수준을 분석하여 "보완이 필요한 의견서입니다"라고 기관 담당자에게 미리 가이드해주는 기능도 고려해볼 만합니다.
3.	멘토 수당 증빙 자동화: 멘토링 의견서 내에 상담 일시, 내용, 참여자 캡쳐 화면 등을 자동으로 포함하여 PDF로 생성해주면, 기관 담당자의 서류 업무가 90% 이상 줄어들 것입니다.
이 확장된 방향으로 PRD의 상세 기능 명세(API 및 화면 설계)를 구체화해 드릴까요? 특히 멘토가 이력서와 통장사본을 업로드하는 보안 스토리지 설계나, 기관 담당자의 수당 지급 승인 워크플로우를 먼저 다뤄볼 수 있습니다.
