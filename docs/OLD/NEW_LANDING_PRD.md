# CASA 신규 랜딩 페이지 PRD

> **CASA** = **C**omprehensive **A**I **S**tartup **A**ssistant (종합 AI 창업 어시스턴트)
>
> **목표:** CASA를 대한민국 예비창업자·창업자 누구나 사용할 수 있는 **독립 AI 창업 플랫폼**으로 포지셔닝하고, 두 가지 트랙(예비창업자/창업자)의 가치를 명확하게 전달하여 회원가입 전환을 극대화하는 랜딩 페이지
> **문서 버전:** 2.0
> **작성일:** 2026-02-25
> **기준 문서:** PRD.md v1.6

## 1. 리브랜딩 방향

### 1.1 기존 vs 신규 포지셔닝

| 항목 | 기존 (v1.0) | 신규 (v2.0) |
|------|-------------|-------------|
| **브랜딩** | 충북대학교 창업지원단 공식 AI 플랫폼 | **CASA — Comprehensive AI Startup Assistant** |
| **타겟** | 충북대 소속 학생/입주기업 | 대한민국 모든 예비창업자 + 기 창업자 |
| **핵심 가치** | 아이디어→사업계획서 30분 | **두 가지 트랙으로 창업의 모든 단계를 AI와 함께** |
| **톤앤매너** | 학교 공식 플랫폼 (권위적) | 친근하면서도 전문적인 SaaS (신뢰+친밀) |
| **CTA** | "무료로 시작하기" (학생 대상) | **"지금 무료로 시작하기"** (범용 대상) |
| **차별화** | 단일 트랙 (아이디어→문서) | **이중 트랙** (예비창업자 + 창업자) |
| **신규 기능** | — | PDF 업로드, 마크다운 렌더링, 보고서 내보내기 |

### 1.2 브랜드 메시지 체계

```
풀네임:    CASA — Comprehensive AI Startup Assistant (종합 AI 창업 어시스턴트)
슬로건:    "AI와 함께, 창업의 첫 걸음부터 다음 단계까지"
태그라인:  "아이디어 검증부터 사업 전략 수립까지, 당신의 종합 AI 창업 어시스턴트"
엘리베이터: "CASA는 예비창업자에게는 아이디어를 사업계획서로,
            창업자에게는 사업계획서를 전략 보고서로 바꿔주는
            종합 AI 창업 어시스턴트입니다."
```

## 2. 타겟 사용자 재정의

### 2.1 주요 타겟

| 순위 | 타겟 | 설명 | 핵심 니즈 | 연결 트랙 |
|------|------|------|-----------|-----------|
| 1차 | **예비창업자** | 아이디어는 있으나 사업화 경험이 없는 개인/팀 | 아이디어 검증, 사업계획서 작성, 빠른 프로토타이핑 | 예비창업자 트랙 |
| 2차 | **초기 창업자** | 사업을 시작했으나 체계적 전략이 부족한 창업자 | 사업 진단, 전략 수립, 보고서 작성 | 창업자 트랙 |
| 3차 | **창업 동아리/팀** | 대학·커뮤니티 기반 창업팀 | 빠른 문서화, 팀 내 합의, 경진대회 준비 | 예비창업자 트랙 |
| 4차 | **1인 창업자** | 리소스가 제한된 솔로 프리너 | 컨설팅 대체, 비용 절감, 올인원 도구 | 양쪽 트랙 |

### 2.2 사용자 페르소나

**페르소나 A: 아이디어형 예비창업자 — "김민수" (28세)**
- 스타트업에 관심 있는 직장인, 퇴근 후 사이드 프로젝트 구상 중
- 아이디어는 있지만 사업계획서 작성법을 모름
- "내 아이디어가 정말 될까?" 확신이 없음
- → 예비창업자 트랙: 아이디어 입력 → AI 평가 → 사업계획서 생성

**페르소나 B: 초기 창업자 — "이서연" (34세)**
- 1년 된 스타트업 대표, 사업계획서는 있으나 성장 전략이 막막
- 투자 유치를 위한 체계적 사업 분석이 필요
- 외부 컨설팅은 비용 부담
- → 창업자 트랙: 사업계획서 업로드 → AI 검토 → 진단 → 전략 → 보고서

**페르소나 C: 창업 동아리 팀장 — "박지호" (22세)**
- 대학 창업 경진대회 2주 전, 아이디어는 있으나 제출 서류 미완성
- 팀원 3명의 의견을 객관적으로 정리하고 싶음
- → 예비창업자 트랙: 빠른 문서화 + AI 다면 평가

## 3. 핵심 목표 및 성공 지표

### 비즈니스 목표
1. 런칭 3개월 내 가입자 **1,000명** 달성
2. 회원가입 전환율 **5% 이상**
3. 가입 후 프로젝트 생성 비율 **40% 이상**
4. 평균 페이지 체류 시간 **2분 30초 이상**

### KPI

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 회원가입 전환율 | 5%+ | CTA 클릭 → 가입 완료 |
| 페이지 이탈률 | 40% 이하 | GA4 |
| 평균 체류 시간 | 2분 30초+ | GA4 |
| CTA 클릭률 | 15%+ | 이벤트 트래킹 |
| 스크롤 완주율 | 45%+ | Intersection Observer |
| 트랙 선택 비율 | 예비 60% / 창업 40% | 프로젝트 생성 데이터 |

## 5. 섹션별 상세 요구사항

### 5.1 네비게이션 바

**목적:** 브랜드 인지 + 빠른 탐색 + 전환 유도

**레이아웃**
- 상단 고정 (sticky), 스크롤 시 배경 블러
- 좌측: CASA 로고 + 텍스트 (대학 로고 제거)
- 중앙: 섹션 앵커 (트랙 소개 | 기능 | 시작하기 | FAQ)
- 우측: 언어 전환 (KO/EN) + 다크모드 토글 + 로그인 (ghost) + 시작하기 (primary)
- 모바일: 햄버거 메뉴

**콘텐츠**
```
[CASA 로고]  트랙 소개 | 기능 | 시작하기 | FAQ     [🌐] [🌙] [로그인] [시작하기]
```

**디자인 가이드**
- 높이: 64px
- 배경: 투명 → 스크롤 시 `bg-background/80 backdrop-blur-md`
- 로고: 그라디언트 텍스트 (university 느낌 제거, 모던 SaaS 느낌)
- 시작하기 버튼: `bg-primary text-primary-foreground` rounded-full

### 5.2 히어로 섹션

**목적:** 3초 내 "두 가지 트랙"의 가치를 전달하고, 자신에게 맞는 트랙으로 즉시 안내

**레이아웃**
- 전체 화면 높이 (`min-h-screen`)
- 중앙 정렬, 텍스트 중심 (좌우 분할 아닌 중앙 집중)
- 하단에 2개 트랙 선택 카드
- 배경: 미묘한 그리드 패턴 + 그라디언트 오브

**콘텐츠**
```
뱃지: "Comprehensive AI Startup Assistant"

헤드라인:
"당신의 창업,
 AI가 함께합니다"

서브헤드라인:
"아이디어 검증부터 사업 전략 수립까지.
 당신의 종합 AI 창업 어시스턴트, CASA가 다음 단계를 열어줍니다."

트랙 선택 카드 (2개, 가로 나열):
┌─────────────────────────────────┐  ┌─────────────────────────────────┐
│  💡 아이디어가 있어요            │  │  📋 사업을 운영하고 있어요       │
│                                 │  │                                 │
│  아이디어를 입력하면 AI가        │  │  사업계획서를 업로드하면 AI가    │
│  사업계획서까지 만들어드립니다.   │  │  진단하고 전략을 제안합니다.     │
│                                 │  │                                 │
│  아이디어 입력 → AI 평가         │  │  사업계획서 → AI 검토            │
│  → 문서 생성 → 배포              │  │  → 진단 → 전략 → 보고서         │
│                                 │  │                                 │
│  [예비창업자로 시작하기 →]       │  │  [창업자로 시작하기 →]           │
└─────────────────────────────────┘  └─────────────────────────────────┘

하단 신뢰 요소 (인라인):
✓ 완전 무료   ✓ 회원가입 30초   ✓ 첫 결과 30분 내 확인
```

**디자인 가이드**
- 뱃지: `bg-primary/10 text-primary` pill 형태
- 헤드라인: 52px→60px 볼드, `tracking-tight`, 그라디언트 텍스트 (primary → accent)
- 서브헤드라인: 20px, `text-muted-foreground`, `max-w-2xl mx-auto`
- 트랙 카드: `bg-card border rounded-2xl p-8`, 호버 시 `border-primary shadow-lg scale-[1.02]`
- 좌측 카드: primary 계열 아이콘/악센트
- 우측 카드: secondary/green 계열 아이콘/악센트
- 배경: 미묘한 도트 그리드 (opacity 3%) + 상단에 큰 그라디언트 원형 blob (primary/5)
- 트랙 카드 내부에 미니 플로우 다이어그램 (아이콘으로 표현)
- 애니메이션: 헤드라인 fade-up → 서브헤드라인 fade-up → 카드 2장 동시 fade-up (stagger)

### 5.13 FAQ 섹션

```
Q1. CASA는 무료인가요?

Q2. 예비창업자 트랙과 창업자 트랙의 차이가 뭔가요?
A. 예비창업자 트랙은 아이디어를 입력하면 AI가 검증·평가·문서 생성까지 해줍니다.
   창업자 트랙은 기존 사업계획서를 바탕으로 AI가 검토·진단·전략·보고서를 만들어줍니다.
   프로젝트 생성 시 트랙을 선택할 수 있습니다.

Q3. 어떤 종류의 아이디어/사업이든 가능한가요?
A. 네, IT 서비스, 제조업, 서비스업 등 분야에 관계없이 모든 창업 아이디어와
   사업계획서를 입력할 수 있습니다. AI가 해당 분야에 맞춰 분석합니다.

Q4. AI가 생성한 문서의 품질은 어떤가요?
A. 사업계획서는 정부지원사업 양식을 준수하며 실제 제출 가능한 수준입니다.
   보고서는 전문 컨설턴트 수준의 분석을 제공합니다.
   또한 각 단계에서 사용자가 직접 검토·수정할 수 있어 원하는 품질을 달성할 수 있습니다.

Q5. AI가 모든 것을 자동으로 결정하나요?
A. 아닙니다. CASA는 'Human-in-the-Loop' 방식으로, AI는 초안을 만들고
   모든 중요 결정은 사용자가 검토·승인합니다. 단계마다 수정과 재생성이 가능합니다.

Q6. 사업계획서 PDF를 업로드할 수 있나요?
A. 네, 창업자 트랙에서 PDF 파일을 업로드하면 텍스트가 자동으로 추출됩니다.
   최대 10MB까지 지원하며, 추출된 텍스트를 직접 수정할 수도 있습니다.

Q7. 결과물을 다운로드할 수 있나요?
A. 네. 예비창업자 트랙에서는 사업계획서를 PDF/Word로 다운로드할 수 있고,
   창업자 트랙에서는 보고서를 인쇄하거나 Word 문서(.doc)로 다운로드할 수 있습니다.

Q8. 내 아이디어와 사업 정보의 보안은 안전한가요?
A. 모든 데이터는 암호화되어 저장되며, Row Level Security를 통해
   본인만 접근할 수 있습니다. 입력 데이터가 외부에 노출되거나
   AI 학습 데이터로 사용되지 않습니다.

Q9. 영어로도 사용 가능한가요?
A. 네, 한국어와 영어를 지원합니다. 언어 설정에서 전환 가능하며,
   AI 생성 문서도 선택한 언어로 작성됩니다.

Q10. 팀원과 함께 사용할 수 있나요?
A. 현재 MVP에서는 개인 계정 기반으로 운영됩니다.
   결과물을 공유 링크나 문서 다운로드로 팀원과 공유할 수 있습니다.
   향후 팀 협업 기능이 추가될 예정입니다.
```

- **Tailwind CSS** + **shadcn/ui**
- **next-intl** (한국어/영어)
- **next-themes** (다크모드)
- **lucide-react** (아이콘)

### 7.3 다국어 (i18n)

`landing` 네임스페이스 전면 재작성:

### 7.4 성능 요구사항

| 지표 | 목표 |
|------|------|
| LCP (Largest Contentful Paint) | 2.5초 이하 |
| Lighthouse 성능 점수 | 90점 이상 |
| CLS (Cumulative Layout Shift) | 0.1 이하 |
| FID (First Input Delay) | 100ms 이하 |

- 이미지: Next.js Image 최적화, WebP/AVIF
- 폰트: `next/font`로 최적 로딩
- 뷰포트 밖 섹션: lazy loading (`Intersection Observer`)
- 번들: 클라이언트 컴포넌트 최소화

### 7.5 접근성

- WCAG 2.1 AA 준수
- 모든 인터랙티브 요소 키보드 접근 가능
- 적절한 heading 계층 (h1 → h2 → h3)
- 색상 대비 4.5:1 이상
- `prefers-reduced-motion` 존중
- 스크린 리더 호환 (aria-label, role)

### 7.6 반응형 디자인

```
데스크톱 (1200px+):
  컨테이너: max-w-7xl mx-auto
  그리드: 2~3열
  트랙 카드: 가로 2열

태블릿 (768px ~ 1199px):
  그리드: 2열
  트랙 카드: 가로 2열 (축소)

모바일 (767px 이하):
  그리드: 1열
  트랙 카드: 세로 스택
  CTA 버튼: 전체 폭
  네비게이션: 햄버거 메뉴
```

### 7.7 애니메이션

```
스크롤 트리거:
  - Intersection Observer 기반
  - fade-up (translateY 20px → 0, opacity 0 → 1)
  - 300ms ease-out
  - stagger: 그룹 내 아이템 150ms 간격

카운트업:
  - 숫자 섹션에 requestAnimationFrame 기반
  - 0 → 목표값, 1.5초 duration
  - ease-out 커브

호버:
  - 카드: translateY(-2px), shadow-md
  - 버튼: scale(1.02), shadow-lg
  - 전환: transition-all 200ms~300ms

다크 섹션 (AI 기술):
  - 연결선 흐르는 애니메이션 (CSS animation, 3초 loop)
  - 글로우 효과 (box-shadow pulse)
```

## 8. CTA 전략

### 8.1 CTA 배치 (총 6곳)

| 위치 | CTA | 타입 |
|------|-----|------|
| 네비게이션 | "시작하기" | 항상 노출, 단일 버튼 |
| 히어로 | "예비창업자로 시작하기" / "창업자로 시작하기" | 트랙 선택 카드, 메인 전환점 |
| 사용자 후기 하단 | "나도 시작하기 →" | 중간 전환 |
| 프로세스 하단 | "지금 무료로 시작하기 →" | 행동 촉구 |
| 최종 CTA | "예비창업자로 시작하기" / "창업자로 시작하기" | 트랙 선택, 최종 전환 |
| 각 트랙 상세 하단 | "이 트랙으로 시작하기 →" | 트랙별 전환 |

### 8.2 전환 흐름

```
CTA 클릭
  → /signup 페이지 이동
  → 회원가입 완료
  → 대시보드 (/dashboard)
  → "새 프로젝트" 클릭
  → 트랙 선택 (예비창업자 / 창업자)
  → 프로젝트 시작
```

히어로/최종 CTA에서 트랙을 선택한 경우, signup 완료 후 해당 트랙의 프로젝트 생성으로 바로 안내하는 것이 이상적입니다. (URL 파라미터: `/signup?track=pre_startup` 또는 `/signup?track=startup`)

### 8.3 심리적 저항 제거

| 저항 요소 | 제거 전략 | 배치 위치 |
|-----------|-----------|-----------|
| 비용 부담 | "완전 무료" 반복 강조 | 히어로, 프로세스, 최종 CTA |
| AI 불신 | "Human-in-the-Loop" — 사용자가 최종 결정 | FAQ, 트랙 상세 |
| 시간 부담 | "30분이면 첫 결과" | 히어로, 숫자, 프로세스 |
| 품질 우려 | "전문가 수준", "실제 제출 가능" | FAQ, 후기 |
| 보안 우려 | "암호화 저장, 본인만 접근" | FAQ |
| 선택 어려움 | "어떤 트랙이 맞는지 모르겠다면?" 가이드 | 이중 트랙, FAQ |
