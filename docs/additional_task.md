## 버그 수정

### BUG-01. 프로젝트 목록 중복 카드 현상
- **현상:** 새 프로젝트 추가 시 목록에 중복 카드가 나타남
- **작업:**
  - [ ] 프로젝트 생성 버튼 — 응답 수신 전까지 비활성화 처리
  - [ ] 프로젝트 생성 API — 멱등성 보장 (단시간 중복 요청 차단)
  - [ ] 프로젝트 목록 상태 — API 응답 완료 후 서버 데이터로 단일 갱신
- **위치:** `app/api/patents/` (또는 프로젝트 생성 라우트), 대시보드 목록 컴포넌트

---

### BUG-02. 계획된 도면 수와 실제 생성 수 불일치
- **현상:** STEP2에서 3개 계획 → STEP5에서 더 많은 도면 생성됨
- **작업:**
  - [ ] STEP5 도면 생성 API — STEP2 계획 데이터(`planned_count`, 도면 목록) 참조 로직 추가
  - [ ] Gemini 프롬프트 — `"정확히 N개의 도면만 생성하시오"` 지시 추가
  - [ ] STEP5 UI — 진입 시 계획된 도면 목록을 사용자에게 표시
- **위치:** `app/api/patents/[id]/drawings/generate/route.ts`, `features/patent/step5/`

---

### BUG-03. 도면 재생성 시 진행 상태 미표시
- **현상:** 재생성 버튼 클릭 후 아무 피드백 없음 (진행 중인지 알 수 없음)
- **작업:**
  - [ ] 재생성 버튼 — 클릭 시 로딩 스피너 표시 및 중복 클릭 방지
  - [ ] 재생성 API — SSE 스트리밍(`generating` / `done` / `error` 이벤트) 방식으로 전환
  - [ ] 도면 카드 UI — 재생성 중 진행 인디케이터 표시
- **위치:** `app/api/patents/[id]/drawings/[drawingId]/regenerate/route.ts`, `features/patent/step5/`

---

### BUG-04. 도면 이미지 하단 잘못된 Fig. 번호 삽입
- **현상:** 생성된 도면 이미지 하단에 Fig. 번호 텍스트가 포함됨
- **작업:**
  - [ ] Gemini 도면 생성 프롬프트 수정 — `"이미지 내부에 Fig. 번호, 제목, 텍스트 라벨을 포함하지 마시오"` 지시 추가
  - [ ] `patentai_prompts` 테이블의 도면 생성 프롬프트 항목 업데이트
- **위치:** `app/api/patents/[id]/drawings/generate/route.ts`, 관리자 프롬프트 관리 페이지

---

### BUG-05. 도면 이미지에 "KIPO 도상" 문구 오삽입
- **현상:** 생성된 도면에 "KIPO 도상" 라벨이 간헐적으로 추가됨
- **작업:**
  - [ ] Gemini 도면 생성 프롬프트 수정 — `"워터마크, 기관명, 'KIPO', '특허청', 도장, 로고 등 일체의 기관 식별 요소를 포함하지 마시오"` 지시 추가
  - [ ] (선택) 생성 후 이미지 텍스트 검출로 키워드 포함 시 자동 재시도 (최대 2회)
- **위치:** `app/api/patents/[id]/drawings/generate/route.ts`

> **참고:** BUG-04와 BUG-05는 동일 프롬프트 수정으로 함께 처리 가능

## 신규 기능

### FEAT-01. 외부 도면 업로드 및 AI 해석/재생성
- **요구사항:** 사용자가 외부 기술 도면을 업로드하고, AI가 해석하거나 특허 규격으로 변환
- **작업:**
  - [ ] **업로드 UI** — STEP5 도면 탭에 "직접 업로드" 옵션 추가 (PNG/JPG/SVG/PDF 지원)
  - [ ] **DB 저장** — `patentai_patent_drawings` 테이블에 `source: 'external'` 플래그 컬럼 추가, Supabase Storage에 원본 이미지 보관
  - [ ] **AI 해석 API** — GPT-4o Vision 또는 Gemini Vision으로 도면 분석 → 구성요소/참조번호/동작 흐름 추출
  - [ ] **명세서 자동 반영** — 추출된 설명을 S7(도면의 간단한 설명), S8(발명 실시 내용)에 반영 제안
  - [ ] **간소화 재생성** — 업로드된 도면을 특허 규격(흑백 선화) 스타일로 Gemini가 재생성, 원본 vs 재생성 비교 표시
- **위치:**
  - `features/patent/step5/` — 업로드 및 비교 UI 컴포넌트
  - `app/api/patents/[id]/drawings/external/route.ts` — 외부 도면 처리 API (신규)
  - `lib/ai/` — Vision 분석 래퍼 함수
